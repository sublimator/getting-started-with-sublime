Introduction

Sublime is extremely configurable. As mentioned in the command section, nothing is hard coded. Every keybinding is customizable as are all of the drop down menus. The theme is customizable. Every thing is configured by editing plain text files, some structured in xml, others in a very simple ini type format.

Opening Warning

It is HIGHLY, SUPER STRONGLY recommended to put your data directory under version control. Commit before every update of Sublime. At the time of writing the Package upgrade rules don't account for user modifications to packages that ship with Sublime. eg changes to assets in the Python package that comes with Sublime.

It is therefore possible, however very unlikely, to lose your customized settings inbetween updates. This is far from ideal but not really that big of a deal as long as safe configuration is practiced. There are some best practices you can follow to minimize the risk.

Some of the settings can be placed in a "User" package which is automatically merged with defaults but in some cases it's impossible to avoid having to manually merge in changes.

This is not something that you'll need to do often but it pays to be aware.

Try this

  • Open the python console
  • Type in and evaluate the following python
  • import sublime

# Find the Data directory
print sublime.packagesPath()

# Find where user installed packages go
print sublime.installedPackagesPath()
